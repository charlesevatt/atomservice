Introduction

The configuration file associated with a specific Atom Service instance dictates what, and how often the service will actually do.

As such it is very important to get your configuration right for the needs of your system.

Variables configurable in an Atom Service configuration file

Variable name Data type Description Typical value(s)
POLL_INTERVAL Integer Interval in seconds between process list polls. The daemon sleeps for POLL_INTERVAL seconds once all processes are executed. Set this to a non-negative value, or set to zero to disable sleeping. 30
PROC_INTERVAL Integer Similar to POLL_INTERVAL, this value specifies the duration in seconds of inter-process sleep 5
AUDIT_LEVEL Constant from Python logging module Logging level DEBUG, INFO, WARNING, ERROR or CRITICAL
NAME String Name of the created daemon to the host system. Shows in the process list. Any name
LOGFOLDER String Specifies folder in which to store log files. New in v.1.0.1 Any valid, writable path
PIDFOLDER String Specifies folder in which to store PID files. New in v.1.0.1 Any valid, writable path
PROC_LIST Array

Contains the list of processes to execute in sequence. First element contains the name you wish the created process to be called and the path to the script to execute, and the second element is an array which can contain a set of arguments to pass to the script executed.

n.b. If you include a direct path to a script as the process to execute, the script must have executable permissions.

Include a line such as:

PROC_LIST.append(Exe('processname', os.path.abspath('myprocess.py'), ['somedata', 12345]) )

Leave the second argument blank if no arguments to be passed e.g.:

PROC_LIST.append(Exe('processname', os.path.abspath('myprocess.py'), ['']) )
